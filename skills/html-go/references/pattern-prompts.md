# Pattern Prompts

Use these as internal generation briefs after selecting a scenario. Adapt labels to the source language.

## 1. Three Code Approaches

Create an HTML comparison page with a compact header, three side-by-side approach cards, tradeoff chips, a decision matrix, risk callouts, and a final recommendation. Include code snippets only when they clarify the difference.

## 2. Visual Design Directions

Create a design exploration page with 3-4 rendered direction panels. Each panel should show layout rhythm, palette swatches, type treatment, interaction notes, and best-fit use cases.

## 3. Annotated PR Review

Create a PR review page with repo/PR metadata, summary bullets, risk map chips linking to file sections, file cards, dark diff blocks, inline review comments, collapsed safe files, and a final checklist.

## 4. Module Map

Create a module understanding page with a large inline SVG architecture map, entry points, hot path, dependency table, risky boundaries, and "where to start reading" guidance.

## 5. Living Design System

Create a design-token page with color swatches, typography scale, spacing/radius tokens, component examples, and copyable token values if useful.

## 6. Component Variants

Create a component contact sheet with variants grouped by size, intent, state, density, and accessibility notes. Use realistic component text and compact labels.

## 7. Animation Sandbox

Create a single-file motion sandbox with a preview element, controls for duration/easing/distance or relevant parameters, live value readout, and implementation notes.

## 8. Clickable Flow

Create a click-through prototype with 3-5 screens, previous/next or hotspot navigation, current-step indicator, and notes on key UX decisions.

## 9. Arrow-Key Slide Deck

Create a slide deck as one HTML file. Use full-screen-ish sections, left/right keyboard navigation, progress dots/count, and concise slide content. Keep each slide within the desktop presentation viewport; split overloaded content into additional slides instead of relying on vertical scrolling.

## 10. SVG Figure Sheet

Create a figure sheet with multiple inline SVG illustrations, captions, usage notes, and a consistent visual vocabulary. Keep each figure copyable as a standalone SVG block when practical.

## 11. Weekly Status

Create a weekly status report with headline summary, shipped/slipped/next sections, small inline chart or metric strip, blocker/risk table, and owners.

## 12. Incident Timeline

Create an incident report with impact summary, minute-by-minute timeline, detection/response/recovery phases, root cause, evidence/log excerpts, and follow-up checklist.

## 13. Annotated Flowchart

Create an annotated process page with an inline SVG flowchart, clickable steps or hover/focus details, timing/failure path notes, and a legend. Keep every node label readable at narrow widths; stack dense node groups or use auto-fit grids instead of compressed columns.

## 14. Feature Explainer

Create a feature explainer with TL;DR, request/data path, collapsible steps, tabbed snippets/config examples if useful, glossary, and FAQ.

## 15. Concept Explainer

Create a teaching artifact with a visual model, interactive controls when helpful, comparison table, worked examples, glossary, and misconceptions. For concept diagrams with long labels or code-like names, include shrink-safe grid children and explicit wrapping rules.

## 16. Implementation Plan

Create a handoff-ready implementation plan with summary metrics, numbered milestones, data flow diagram, lightweight mockups, key code/contracts, risk table, and open questions. Check that nested cards, flow nodes, and prompt/code panels stay inside their parent borders on desktop and mobile.

## 17. PR Writeup

Create a reviewer-facing PR writeup with motivation, before/after, file-by-file tour, testing evidence, review focus areas, and known non-goals.

## 18. Ticket Triage Board

Create a kanban-style triage board with columns such as Now/Next/Later/Cut, item cards, estimates, tag filters, and a "Copy as markdown" export.

## 19. Feature Flag Editor

Create a grouped flag editor with toggles, rollout labels, dependency warnings, dirty-state highlighting, reset, and copy/export diff.

## 20. Prompt Tuner

Create a prompt tuning interface with editable template, highlighted variables, sample input cards, live preview outputs, and a copy final prompt button.
