01

项目初衷

Markdown 仍然适合记录和长期保存。HTML Go 关注的是另一件事：当输出需要被展示、比较、点击、演示或讨论时，HTML 是更好的交付格式。

Why This Exists

Markdown is still great for capture and memory. HTML Go focuses on the other half: when the output needs to be presented, compared, clicked, demonstrated, or discussed.

02

HTML 比 Markdown 强在哪里

不是所有输出都需要 HTML。但这些类型的工作，HTML 会明显提升可读性和可操作性。

Where HTML Beats Markdown

Not every answer should become HTML. These work types benefit from the browser as the medium.

Compare

比较和决策

多方案并排、风险矩阵、推荐路径，比连续段落更容易讨论。

Review

代码评审

差异、注释、严重等级和跳转链接可以在一个页面里组织。

Explain

图表和解释器

流程图、状态图、可折叠步骤和 tabs 能降低理解成本。

Present

演示材料

单文件 slide deck 可直接用键盘翻页，不需要导出流程。

Prototype

交互原型

动画、点击流和可调参数只有在浏览器里才能真正被感受。

Edit

临时编辑器

看板、feature flag、prompt tuner 可以最后导出 Markdown 或 JSON。

Compare

Options, criteria, risk matrices, and recommendations are easier to discuss side by side.

Review

Explain

Present

Prototype

03

它如何工作

HTML Go 先判断内容场景，再选择合适结构，最后生成和校验单文件 HTML。

How It Works

HTML Go identifies the source scenario, chooses a fitting structure, then writes and validates a standalone HTML file.

Markdown

适合保留的部分

  • 短回复、代码片段、日志记录
  • 长期知识库和版本控制友好文本
  • 作为 HTML artifact 的输入和导出格式
HTML Go

升级成页面的部分

  • 需要横向比较和视觉层级的交付物
  • 需要图表、状态、演示和互动的内容
  • 需要在会议或评审中直接打开的成品
Markdown

Keep it for

  • Short replies, snippets, and logs
  • Long-term notes and version-controlled text
  • Input and export formats for artifacts
HTML Go

Upgrade when

  • The deliverable needs layout and visual hierarchy
  • The content needs charts, state, slides, or interaction
  • The result should open directly in a meeting or review
04

安装和使用

不同工具读取扩展的方式不同：Codex 和 Claude Code 使用 skill 目录，Gemini CLI 使用 slash command。

Install and Use

Different tools load extensions differently: Codex and Claude Code use skill directories, while Gemini CLI uses a slash command.

# Codex
git clone git@github.com:zacktian89/html-go-skill.git
cd html-go-skill
New-Item -ItemType Directory "$env:USERPROFILE\.codex\skills" -Force
Copy-Item .\skills\html-go "$env:USERPROFILE\.codex\skills\html-go" -Recurse -Force
# Claude Code
git clone git@github.com:zacktian89/html-go-skill.git
cd html-go-skill
New-Item -ItemType Directory "$env:USERPROFILE\.claude\skills" -Force
Copy-Item .\skills\html-go "$env:USERPROFILE\.claude\skills\html-go" -Recurse -Force
# Gemini CLI
New-Item -ItemType Directory "$env:USERPROFILE\.gemini\commands" -Force
# 创建 html-go.toml 自定义命令
# 然后使用 /html-go 生成单文件 HTML

HTML Go 保持一个克制的边界：Markdown 负责轻便记录，HTML 负责真正需要被打开、阅读和操作的交付物。

HTML Go keeps a narrow boundary: Markdown for lightweight memory, HTML for deliverables that need to be opened, read, and used.
