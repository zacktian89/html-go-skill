微信公众号复制粘贴 HTML 测试页

这个页面用于验证 HTML Go 能否扩展出面向公众号编辑器的输出模式：正文区域使用简单标签和内联样式，按钮会把正文以 text/html 写入剪贴板。

测试要点

  • 01标题、导语、引用、列表、表格、代码块是否保留基础样式。
  • 02粘贴进公众号编辑器后，行距、边框、强调色是否接近预期。
  • 03正文是否不依赖外部 CSS、脚本、字体或图片资源。

点击按钮后，到微信公众号后台编辑器中粘贴。若浏览器限制剪贴板权限，可手动选中右侧正文复制。

扩展判断

可以扩展。建议把它作为 HTML Go 的一个独立输出模式，保留现有浏览器 artifact 模式。

原因很直接：浏览器 artifact 可以保留复杂布局和交互；公众号粘贴 HTML 需要更克制，重点是正文结构、内联样式、可复制性和编辑器兼容性。

最小规则可以是：

  • 正文容器内全部关键样式使用 style=""。
  • 避免脚本、外链 CSS、外部字体、复杂 grid/flex。
  • 保留语义标签，但优先使用公众号编辑器更稳的 section、p、span、table。
  • 生成一个复制按钮，把正文作为 text/html 放进剪贴板；含图内容同时尝试写入 image/png。

把 artifact 变成公众号可粘贴正文

这是一段用于测试的导语。目标是生成一段粘贴到公众号编辑器后仍然可读、可改、样式不散的正文 HTML。

一句话结论

HTML Go 可以增加一个 WeChat Paste 模式：输出一个测试页，同时内置可复制的公众号正文 HTML。

它和普通 HTML artifact 的差异

普通 artifact 追求完整的浏览器体验，可以使用响应式布局、交互、视觉层级和局部脚本。公众号粘贴模式追求的是编辑器兼容性，因此样式需要收敛。

项目 建议
样式 关键样式放到内联 style
布局 单列正文优先，少用复杂嵌套
交互 仅保留复制按钮，正文不依赖 JS

推荐的最小输出结构

  1. 外层测试页：包含预览、说明、复制按钮。
  2. 正文片段：只保留公众号要粘贴的内容。
  3. 剪贴板写入：同时提供 text/html 和纯文本兜底。
